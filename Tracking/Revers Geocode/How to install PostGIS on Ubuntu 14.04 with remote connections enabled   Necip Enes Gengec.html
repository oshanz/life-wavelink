How to install PostGIS on Ubuntu 14.04 with remote connections enabled

Home / PostGIS / How to install PostGIS on Ubuntu 14.04 with remote connections enabled
28/08/2014 by Necip

I know this sounds simple. As a mapping engineer with too few IT and programming background, this kind of applications may be too difficult to handle. I know this, because it was really hard for me to figure out these. So that I thought documenting the all process can be helpful for me and for others.

Firstly I’d like to introduce my motivation and I will try to share the rest:

I am trying to setup a web server which I will

  • have a spatial database that I can connect remotely,
  • have a map server (Geoserver in my case),
  • and develop a web application to serve and manipulate mapping data.

This is more or less the same as in the opengeo suite (refer to below diagram in the link). So the one can connect to the remote database and upload and manage data and database (PostGIS/Postgresql), manipulate data and use the data with connecting to this remote database with applications (QGIS) and develop web applications and serve maps (Geoserver/Openlayers).

In this article I will explain how to setup a Postgresql database in a remote Ubuntu 14.04 server with PostGIS and how to connect this database using pgAdmin and QGIS.

First I bought a host over Digitalocean with the 1GB memory and 30GB disk space with Ubuntu 14.04 LAMP installation. I would suggest you to use at least 1GB memory options in your host provider since Geoserver crashes frequently with less memory.

After you need to connect to your host and add Postgreql repository to your source list, update and install necessary packages. (Sound crazy right?) Run the following commands in your ssh command line:

sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt trusty-pgdg main" >> /etc/apt/sources.list'
sudo apt-get update
sudo apt-get install postgresql-9.3 pgadmin3 postgresql-contrib
sudo apt-get install postgresql-9.3-postgis-2.1 pgadmin3 postgresql-contrib

These will install the Postgresql 9.3 with the PostGIS extension. Please change the version numbers in the commands if you try to install a newer version.

After installing the Postgresql and PostGIS without an error, you need to connect to Postgresql database and create some extensions. These are Adminpack (needed for some functions of pgAdmin), PostGIS, PostGIS topology and PostGIS Tiger geocoder. The fuzzystrmatch seems not required in the references but I had and error about it, so I also added to the below code. Please follow the below commands. These will connect to Posgresql with postgres user and creates above mentioned extensions.

sudo su - postgres
CREATE EXTENSION adminpack;
CREATE EXTENSION fuzzystrmatch;
CREATE EXTENSION postgis;
CREATE EXTENSION postgis_topology;
CREATE EXTENSION postgis_tiger_geocoder;
\q

At this point I will explain how to create a Postgresql database user. I am sure this may be unnecessary and there may be other solutions but I am suggesting you to have another user that postgres. Please follow below commands. Change “yourUserName” part into your user name and it will ask you for your password and second time again for confirmation. In my case I created a user called “necipgisserver” use this in the further examples.

sudo su - postgres
createuser -d -E -i -l -P -r -s yourUserName

You can leave the postgres user with typing “exit“. We will use this user name and password for connection.

We are coming to the last part which is enabling the postgresql service to the remote connections. For enabling your database service to the remote connections you need to modify two files:

  • pg_hba.conf file: client authentication
  • postgresql.conf file: postgresql service configuration

In authentication file run the below code. This will open the authentication in nano text editor:

sudo nano /etc/postgresql/9.3/main/pg_hba.conf

Be careful with the Postgresql version and change it into your installed version. When you get in to the file ad the below line to the end of the file. The file will look like the below image. By adding this line you are enabling all users to connect over any IP with the user/password pair.

host   all     all 0.0.0.0/0   md5

You can leave the nano with Ctrl + x. Follow the instructions and save the file.

In Postgresql service configuration file we need to let the service listen to all connections in order to connect over any IP. So we will reach the file using nano and add the below line to the end of the file. Again be careful with the postgresql version.

sudo nano /etc/postgresql/9.3/main/postgresql.conf
listen_addresses = '*'

This is all we need to do with the configuration and authentication files. Please note that this opens everything if someone knows your user name and password can connect easily. You can comment these lines out by adding “#” without to quotes.

All the hard part ended. Now you can connect to your remote server using pgAdmin and QGIS. I will only share the connection screen examples for both platforms. You can easily apply your own configuration reviewing these examples. Using these configurations you can now connect to your remote server like you are connecting on your PC and you can deploy your data using QGIS and pgAdmin and you can run all database tasks.
